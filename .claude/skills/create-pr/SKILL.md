# PR 자동 생성 스킬 (create-pr)

이 스킬은 현재 작업 중인 브랜치의 변경 사항을 요약하고, E2E 테스트 통과 여부를 검증한 후, GitHub CLI(`gh`)를 활용하여 안정적이고 포맷에 부합하는 Pull Request를 자동으로 게시하도록 강제합니다.

## 사용법

- 명시적 호출: `/create-pr [GitHub 이슈 번호]` (예: `/create-pr 1`)
- 자연어 트리거: "PR 만들어줘", "PR 올려줘", "PR 보내줘", "이거 머지하자", "create-pr" 등의 요청 시 자동 실행

## 입력 인자

- `$ARGUMENTS`: GitHub 이슈 번호 (선택 사항이며, 제공된 경우 PR 본문에 이슈 연결 코드가 자동 삽입됩니다)

---

## 📌 상세 실행 단계

이 스킬이 가동되면 다음 5단계를 순서대로 엄격히 수행합니다.

### 1단계: 변경 사항 분석 및 PR 초안 작성

- **동작**:
  - `git status` 및 `git diff`를 실행하여 작업 공간의 수정 및 추가 사항을 면밀히 분석합니다.
  - 이를 요약하여 **PR 제목**과 **PR 본문(Body)** 초안을 Markdown 형식으로 작성합니다.
  - **이슈 종결 구문 주입**: 입력 인자 `$ARGUMENTS`가 있는 경우, 본문 하단에 반드시 **`Closes #$ARGUMENTS`** 구문을 포함하여 머지 시 관련 이슈가 자동으로 닫히도록 연결합니다.
  - **Base 브랜치 자동 탐색**: 자식 브랜치(`feat/`)가 파생된 부모 브랜치(`feature/` 브랜치)를 탐색하여 PR의 Target Base 브랜치 후보로 설정합니다.

### 2단계: 개발자 검토 및 초안 승인

- 생성된 PR 제목, Base 브랜치, 본문 초안을 개발자에게 보여주고 **검토 및 승인**을 요청합니다.
- 개발자가 피드백을 주면 초안을 수정하며, 최종 컨펌이 날 때까지 **대기**합니다.

### 3단계: E2E 테스트 실행 및 검증

- 개발자의 승인을 얻으면, E2E 테스트 검증 단계에 돌입합니다.
- **실행**: `npm run test:e2e` 명령어를 실행하여 Playwright E2E 테스트 전체를 검증합니다.
- **예외 (Pass 허용)**: 만약 해당 이슈가 UI 화면이 없는 순수 백엔드/API 단계여서 **E2E 테스트할 대상이 아직 없는 상황이라면, 이 단계를 건너뛰고(pass) 다음 단계로 진행할 수 있습니다.**

### 4단계: E2E 실패 시 대처 및 복구 지침 (중요)

- 만약 E2E 테스트 중 하나라도 실패하는 경우, **PR 생성을 즉시 중단**하고 개발자에게 에러 요약을 보고한 뒤 아래의 **복구 가이드라인을 준수**하여 다음 단계 처리를 유도합니다.
  - **E2E 실패 복구 프로세스**:
    1. **Trace Viewer 실행**: `npx playwright show-report`를 실행하여 어느 부분의 UI나 시나리오가 깨졌는지 정확히 디버깅합니다.
    2. **원인 분석**: 오류가 API 레이어, UI 렌더링 레이어, 혹은 핵심 비즈니스 로직 레이어 중 어디서 기인했는지 식별합니다.
    3. **단위 테스트 케이스 추가 (Red)**: 식별한 실패 시나리오를 검증하는 단위 테스트를 관련 `*.test.ts/tsx` 파일에 추가하여 테스트가 실패하게 만듭니다.
    4. **프로덕션 코드 수정 (Green)**: 단위 테스트가 통과하도록 소스 코드를 교정합니다. 통과 후 다시 `/create-pr`을 호출합니다.
  - ⚠️ **주의사항**: **절대 E2E 테스트 코드 자체를 인위적으로 고쳐서 테스트를 강제 통과시키지 마십시오.** 이는 근본적인 원인을 회피하는 위험한 우회입니다.

### 5단계: 코드 푸시 및 Pull Request 생성

- E2E 테스트가 100% 정상 통과하면 다음을 수행합니다.
  - **커밋 메시지 검사**: 최종 커밋 메시지가 프로젝트 `commitlint` 포맷(feat, fix, refactor 등 접두사 및 소문자 규칙)에 부합하는지 검사합니다.
  - **Push**: `git push origin [현재브랜치명]`을 수행합니다.
  - **PR 생성**: GitHub CLI를 사용하여 PR을 발행합니다.
    - 커맨드: `gh pr create --title "[PR 제목]" --body "[PR 본문]" --base [부모브랜치명]`
  - **이슈 피드백**: 생성된 PR의 주소(URL)를 획득하여 관련 GitHub 이슈에 PR 링크를 포함하는 완료 댓글(Comment)을 작성하고 전체 스킬의 실행을 마칩니다.
