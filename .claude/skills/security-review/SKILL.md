# 보안 및 타입 검증 스킬 (security-review)

이 스킬은 리팩토링이 완료되고 최종 커밋을 수행하기 직전, 타입 안정성을 확인하고 보안상 치명적인 취약점이나 환경 변수의 소스 코드 노출 여부를 면밀히 스캔하여 교정하는 가이드라인을 강제합니다.

## 사용법

- `/security-review [GitHub 이슈 번호]` (예: `/security-review 1`)

## 입력 인자

- `$ARGUMENTS`: GitHub 이슈 번호 (예: `1`)

## 상세 실행 단계

이 스킬이 활성화되면 다음 단계를 순서대로 엄격히 수행하며, 수집된 잠재적 오류 및 위협 요소들을 체계적으로 분류하여 대처합니다.

### 1단계: TypeScript 타입 검사

- **동작**: `npx tsc --noEmit` 명령어를 실행하여 현재 프로젝트 내에 존재하는 TypeScript 컴파일 타입 에러 목록을 수집합니다.

### 2단계: npm 패키지 취약점 수집

- **동작**: `npm audit` 명령어를 실행하여 사용 중인 서드파티 라이브러리 및 패키지의 보안 취약점 목록을 수집합니다.

### 3단계: 환경 변수(.env) 노출 검사

- **동작**: `.env`에 정의된 민감한 설정 값이나 비밀 키(Secret Key)가 `src/` 디렉토리 내 소스 코드에 하드코딩되거나 문자열 형태로 직접 노출되어 있는지 grep 또는 정규식 검색 도구로 점검합니다.
  - 예: `process.env.XXXX` 또는 환경 변수 문자열 값 자체가 소스 파일에 그대로 적혀 있는 사례 검색

### 4단계: 결과 분류 및 개발자 보고

- 수집된 모든 이슈(타입 에러, npm audit 취약점, env 노출)를 다음 **3가지 등급**으로 구조화하여 개발자에게 명확히 보고합니다.
  - **즉시 수정 필요**:
    - 빌드 오류를 야기하는 TypeScript 타입 에러
    - 소스 코드 내 민감정보 노출(env 변수 하드코딩)
    - npm audit에서 발견된 High/Critical 레벨의 취약점 중 직접 해결 가능한 항목
  - **권장 수정**:
    - 코드 품질을 해칠 수 있는 마이너 에러나 Warning
    - 빌드는 가능하나 잠재적 타입 모호성(any 남용 등)이 존재하는 항목
  - **무시 가능**:
    - 오탐지(False Positive)로 판단되는 항목
    - 의도적으로 허용된 코드 패턴 또는 개발 환경 전용 패키지의 무해한 취약점
- 등급별 항목 리스트를 Markdown 형식으로 요약하여 작성한 후, 개발자의 **승인을 대기**합니다.

### 5단계: 개발자 확인 및 즉시 수정 대상 해결

- 개발자의 컨펌을 얻은 후, **"즉시 수정 필요"**로 분류되고 합의된 이슈들만 코드를 수정하여 해결합니다.
- **제약 사항**:
  - 주관적 판단이나 해결법이 모호한 항목에 대해서는 독단적으로 수정하지 않고, 사전에 개발자와 해결 방법을 반드시 합의하십시오.
  - 테스트 코드 파일(`*.test.ts`, `*.test.tsx`)은 임의로 수정할 수 없습니다.
  - **`npm audit fix --force` 명령어는 종속성 충돌을 일으키므로 절대 실행해서는 안 됩니다.**

### 6단계: 재스캔 및 최종 검증

- 수정 작업을 완료한 후, 1~3단계의 점검 스캔(tsc, npm audit, env grep)을 재실행합니다.
- 모든 위협 요소가 깔끔하게 지워져 클린(Clean) 상태에 도달했는지 확인하고, 최종 확인 결과를 개발자에게 알리며 검증을 마칩니다.
