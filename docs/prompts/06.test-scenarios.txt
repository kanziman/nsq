/skill-creator를 사용해서 아래 조건으로 .claude/skills/test-scenarios/SKILL.md를 만들어줘:

목적: 이슈 단위로 시그니처 확정 → 테스트 시나리오 도출을 순서대로 처리. (참고: 이후 TDD 방법론으로 테스트 코드 구현 예정)

사용법: /test-scenarios 1 (GitHub 이슈 번호)

실행 순서:
  1. 현재 GitHub 이슈 내용과, docs/features/tag/prd.md와 코드베이스를 참고해서 시그니처를 확정한다
     - 함수 시그니처 (이름, 파라미터 타입, 반환 타입)
     - 에러 케이스 (어떤 상황에서 에러를 던지는지)
     - 컴포넌트 Props 타입 정의
     - 구현 코드는 절대 작성하지 않음
     - 기존 패턴을 따를 것 (api/notes.ts, NotesContext, 컴포넌트 Props 패턴)
  2. 확정된 시그니처를 개발자에게 보여주고 검토/승인을 받는다
  3. 승인된 시그니처를 /docs/features/tag/issue-{N}.md 상단에 기록한다
  4. 시그니처를 기반으로 테스트 시나리오를 도출한다
     - 시나리오를 정상/경계/예외로 분류
     - 형식: "[정상/경계/예외] 함수명 — should [기대동작] when [조건]"
     - 테스트 코드는 작성하지 않음
  5. 도출된 시나리오를 /docs/features/tag/issue-{N}.md 하단에 추가한다
  6. GitHub Issue의 AC 항목과 시나리오를 대조한다
     - gh issue view $ARGUMENTS 로 AC 목록을 읽는다
     - AC 항목 중 시나리오가 없는 항목이 있으면 시나리오를 추가한다
     - 모든 AC가 최소 1개 이상의 시나리오로 커버되는지 확인한다
  7. 시나리오를 개발자에게 보여주고 검토/승인을 받는다
     (승인 전까지 다음 단계로 넘어가지 말 것)

기타:
  - 입력: 이슈 번호 ($ARGUMENTS)
