# 에피소드 목록 UI (episodes-list-ui) 확정 요구사항

이 문서는 `spec-original.md`를 바탕으로 인터뷰를 진행하여 합의된 **확정 요구사항 명세서**이다.
이후 개발, 디자인 및 의사결정은 이 문서를 기준으로 진행한다.

---

## 1. 기능 개요

임포트 완료된 에피소드 목록을 홈 화면에서 조회하고 학습 상세 페이지로 진입하며, 신규 추가를 유도하고, 진행 중이거나 실패한 임포트 에피소드들을 모니터링/제어하는 직관적이고 완성도 높은 홈 화면 대시보드 UI를 구축한다.

---

## 2. 핵심 사용자 (Primary User)

- 로컬 환경에서 본인의 쉐도잉 학습을 진행하는 단일 학습자. 본인이 임포트해둔 학습용 에피소드를 한눈에 파악하고 즉시 세션에 진입하고자 하는 사용자.

---

## 3. 확정된 결정 사항

### 3-1. 에피소드 삭제 시 AlertDialog 경고 모달 제공 ✅

- 에피소드 카드의 "삭제(쓰레기통)" 버튼을 누르면 즉시 API를 호출하지 않고, **디자인 시스템의 AlertDialog(경고 다이얼로그) 모달**을 띄운다.
- 문구: `"에피소드를 삭제하시겠습니까? 다운로드된 오디오 및 대본 정보가 컴퓨터에서 영구적으로 삭제됩니다."`
- 사용자가 "삭제"를 최종 승인하면 DELETE API를 호출하고 목록을 갱신한다.

### 3-2. 진행 중 에피소드의 조건부 폴링 (Conditional Polling) ✅

- 홈 화면 진입 시, 받아온 에피소드 목록 중 진행 중(`downloading`, `processing_subtitles`, `processing_transcript`, `aligning`) 상태인 에피소드가 **1개 이상 존재할 때만 3초 주기로 `/api/episodes` GET 요청을 폴링**한다.
- 모든 에피소드가 완료(`completed`) 또는 실패(`failed`) 상태로 전환되면 폴링은 자동으로 멈춘다.

### 3-3. 임포트 실패(`failed`) 에피소드 카드 조작 및 연동 ✅

- 실패한 에피소드 카드에는 눈에 잘 띄는 빨간색 `Failed` 배지를 표시한다.
- 카드 내에 마우스 오버(Hover) 시 구체적인 에러 메시지를 볼 수 있는 **정보 툴팁(Tooltip)**을 제공한다.
- 카드 내에 **"재시도(Retry)"** 버튼을 제공한다. 클릭 시 해당 에피소드의 재시도를 위해 **`/import?videoId={id}`** 경로로 사용자를 이동시킨다.

## 4. 화면 및 컴포넌트 명세

### 4-1. 홈 화면 레이아웃 (`src/app/page.tsx`)

- **디자인 시스템(DESIGN.md) 규격 적용**:
  - 다크/라이트 모드 대응이 가능한 미니멀한 대시보드 레이아웃.
  - 최상단 헤더 영역에 앱 타이틀 및 서비스 안내 메시지.
  - 우측 상단에 새로운 에피소드를 추가하기 위한 `"에피소드 임포트"` 액션 버튼 배치 (클릭 시 `/import` 라우트로 이동).
- **데이터 패칭 처리**:
  - 초기 로딩 시 스켈레톤(Skeleton) 로딩 UI 제공.
  - API 로드 에러 시 깔끔한 에러 상태 카드 및 `"다시 시도"` 버튼 노출.

### 4-2. 에피소드 카드 그리드 컴포넌트 (`src/components/episode-list.tsx`)

- 반응형 그리드 구조 (Grid Layout - Mobile: 1열, Tablet: 2열, Desktop: 3열 이상).
- **개별 에피소드 카드 (EpisodeCard) 세부 디자인**:
  - **썸네일**: YouTube 썸네일 주소 바인딩. 로딩 전이거나 누락 시 기본 대체(Fallback) 그라디언트 카드 이미지 제공.
  - **메타데이터**: 에피소드 제목, 재생 시간(예: `15:42` 또는 `03:20`), 등록일(추가된 날짜 포맷팅) 표시.
  - **진행 중 UI**:
    - 진행 중인 에피소드는 프로그레스 바(Progress Bar)를 카드 내에 표시.
    - 현재 수행 중인 단계(예: "오디오 다운로드 중...", "대본 분석 중...") 텍스트 렌더링.
  - **동작 상호작용 (Interaction)**:
    - 완료(`completed`) 상태 카드 클릭 시 상세 플레이어 `/episodes/[id]` 페이지로 이동.
    - 카드 우측 하단 또는 상단에 개별 "삭제" 액션 버튼 탑재.

## 5. 경계 조건 (Boundary Conditions)

- **빈 에피소드 목록 (Empty State)**: 에피소드가 전혀 등록되지 않은 경우, 휑한 화면 대신 **"등록된 에피소드가 없습니다. 첫 에피소드를 추가해보세요!"** 라는 안내 문구와 함께 큼직한 `"첫 에피소드 임포트하기"` 버튼을 중앙에 크게 배치하여 행동을 유도한다.
- **긴 제목 처리 (Typography)**: 에피소드 제목이 너무 길 경우 카드 레이아웃이 깨지지 않도록 최대 2줄 처리 후 말줄임표(...)로 제한한다.

---

## 6. Ubiquitous Language (용어 정의)

| 용어                                  | 정의                                                                          |
| :------------------------------------ | :---------------------------------------------------------------------------- |
| **빈 화면 (Empty State)**             | 조회 결과가 없을 때 임포트 페이지로 이동을 촉구하는 전용 안내 UI              |
| **조건부 폴링 (Conditional Polling)** | 진행 상태의 잡이 목록에 있을 때만 한시적으로 백엔드를 정기 조회하는 갱신 패턴 |
| **재확인 경고 모달 (AlertDialog)**    | 실수로 에피소드를 지우지 않도록 경고하고 승인받는 인터랙션 컴포넌트           |

---

## 7. 명시적 비범위 (Out of Scope)

- 홈 화면 내에서 에피소드 제목을 검색하는 검색 바(Search Bar) 및 필터링 기능 (MVP 이후 확장)
- 에피소드 카드를 드래그 앤 드롭으로 순서 변경(Reordering)하는 기능
- 홈 화면 내에서 대본 데이터를 즉석 미리보기(Preview)하는 UI 레이어
